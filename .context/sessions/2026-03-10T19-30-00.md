## Checkpoint: 2026-03-10T19:30:00

- **Scope**: Distiller spec revision — session lifecycle, frontmatter conventions, access boundaries, and transcript capture.

- **Since last checkpoint**:
  - Ran first distillation test against `touchy-muchy` domain — produced `.proposed` files manually after skill hit permission prompts (cross-project writes from firehose to touchy-muchy).
  - Identified that the `.proposed` file suffix pattern and `processed/` folder convention were not the right design.
  - Through iterative design discussion, arrived at a revised model:
    - **Sessions are permanent** — never moved or deleted. Lifecycle tracked via YAML frontmatter (`active` → `closed` → `distilled`).
    - **Frontmatter over file suffixes** — status lives inside files, not in filenames. No `.proposed` siblings.
    - **Canonical files are the distilled view** — `MEMORY.md` and `DECISIONS.md` are the distiller's output. Two layers (sessions + canonical), not three.
    - **Checkpoints are subjective** — the agent's curated summary, not objective record.
    - **Distiller is objective** — reads cold artifacts, produces canonical knowledge without attachment.
    - **Re-distillation always possible** — corpus is permanent, distiller is a lens. Swap the lens, re-derive.
    - **Cron as primary trigger** — no dependency on Claude Code being alive. Scans registry for domains with `closed` sessions.
  - Identified session boundary problem: agent can opportunistically flip `status: closed`, unflip if session continues.
  - Identified that the agent's subjective checkpoint and the distiller's objective extraction should coexist with provenance — four disagreement cases documented.
  - Updated `distiller-spec.md` with full revised model.
  - Updated `domain-convention.md` — removed `processed/`, added frontmatter lifecycle, access boundaries, updated post-session flow.
  - Updated `distill-domain` skill — scans frontmatter, writes frontmatter, `--re-distill` support.
  - Updated `checkpoint` skill — writes frontmatter, appends to active session, opportunistic closing.
  - Investigated transcript capture: Claude Code has no native export. No `claudeProcessWrapper` setting exists. `SessionEnd` hook exists but can't access transcript content. Candidates: LiteLLM gateway logging, `script` terminal capture.
  - User escalated: wants full transcripts INCLUDING thinking blocks. This is a major open design question — deferred to its own session.

- **Decisions made**:
  - **Frontmatter over file suffixes**: Status lives in YAML frontmatter, not in filenames (`.proposed`) or folder structure (`processed/`). Enables richer metadata, cleaner filesystem.
  - **Sessions are permanent, distillation is rerunnable**: Session corpus is the durable asset. Distiller is a lens — swap the lens, re-derive. No destructive processing.
  - **Cron as primary distillation trigger**: Decouples distillation from Claude Code process lifecycle. No post-session hook dependency. Signal is on disk: `closed` sessions that aren't `distilled`.
  - **Two layers, not three**: Sessions (subjective) + canonical files (objective distilled view). No separate "distillation layer" needed — canonical files ARE the distillation output.
  - **Transcript capture deferred to dedicated session**: Full transcript + thinking block capture is a foundational concern that needs its own focused design.

- **Open questions**:
  - Transcript capture mechanism: LiteLLM gateway logging vs `script` vs future Claude Code feature. Needs its own session.
  - Does `status: proposed` on canonical files play well with the SessionStart hook? The hook reads MEMORY.md on entry — if it has `status: proposed` frontmatter, the agent would see unreviewed content.
  - Should the cron job also handle approval (auto mode), or only distillation?
  - What happens to the touchy-muchy `.proposed` files from the manual test run? Clean up needed.

- **Current thinking**: The revised model is significantly cleaner. Sessions as permanent corpus with frontmatter lifecycle eliminates the awkward `processed/` folder and `.proposed` file proliferation. The subjective/objective split with provenance is the right framing. The biggest gap is now transcript capture — without full transcripts, the distiller only has the agent's curated checkpoints (subjective) to work with. The `careful` and `adversarial` strategies need richer input.
