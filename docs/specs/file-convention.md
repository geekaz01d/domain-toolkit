# File Convention Spec

**Status:** Draft — captured from working session 2026-03-18
**Context:** Defines the file hierarchy for domain-toolkit managed domains. Supersedes the agent.md convention defined in domain-convention.md. Coexists with the AGENTS.md ecosystem standard.

---

## Design Principles

- **Domain-centric, not agent-centric.** The domain declares itself. Agents adapt to the domain, not the other way around.
- **Coexist with ecosystem conventions.** AGENTS.md and CLAUDE.md are standard hooks that agents read. Domain-toolkit layers on top of them, not instead of them.
- **Governance before persona.** Global default posture loads first. Domain-specific governance next. Persona (role-specific identity) last.
- **Self-propagating defaults.** Global files at `~/` and `~/.claude/` serve as templates. If a repo-level copy is missing, the global version is detected and the repo copy is restored (with a comment noting the fix).

## File Hierarchy

### Global Default Posture (User-Level)

| File | Location | Purpose |
|------|----------|---------|
| `AGENTS.md` | `~/AGENTS.md` | Universal governance. Default posture for any agent in any tool. Session recording discipline, write behaviour, distiller input contract, quality floor. |
| `CLAUDE.md` | `~/.claude/CLAUDE.md` | Claude-specific global instructions. Claude Code reads this as user-level config. |

These are the governance floor. Every agent entering any domain inherits this baseline regardless of toolchain.

### Domain-Level (Repo Root)

| File | Location | Tracked in git | Purpose |
|------|----------|---------------|---------|
| `CLAUDE.md` | `<domain>/CLAUDE.md` | Yes | Domain-specific governance and project mechanics. Build commands, code conventions, spec pointers, workflow. Inherits from and can extend the global default. Claude Code reads this natively. |
| `AGENTS.md` | `<domain>/AGENTS.md` | Yes | Optional. If the domain participates in multi-tool ecosystems. `touch-domain` detects and surfaces its presence. |
| `README.md` | `<domain>/README.md` | Yes | Canonical, human-authored description of what this domain is. |

### Domain-Toolkit Layer (`.claude/domain-toolkit/`)

| File | Location | Tracked in git | Purpose |
|------|----------|---------------|---------|
| `domain.yaml` | `.claude/domain-toolkit/domain.yaml` | Yes | Machine-readable manifest. Registry metadata: name, type, sets, has_services, origin, description. **Detection signal** for managed domains — replaces agent.md's role as the domain identity marker. |

### Persona Layer

| File | Location | Tracked in git | Purpose |
|------|----------|---------------|---------|
| `persona.md` | Contextual — see below | Yes | Context-specific role, rules, and success criteria for an agent operating in a given context. |

**Persona placement is contextual.** `persona.md` goes where you launch an agent from:

- **Domain root** (`<domain>/persona.md`) — default persona for interactive work in this domain
- **Skill directory** (`<domain>/.claude/skills/<skill>/persona.md`) — persona for that skill's agent
- **Subtree** (`<domain>/exec/persona.md`) — scoped to a subdomain or concern within a larger domain
- **Global** (`~/.claude/domain-toolkit/personas/<name>.md`) — globally applicable personas (e.g. a distiller persona, a reviewer persona)

A domain can have zero, one, or many personas. Governance does not depend on persona existing.

### Knowledge Layer (`.context/` — gitignored, synced via Syncthing)

| File | Location | Tracked in git | Purpose |
|------|----------|---------------|---------|
| `PROFILE.md` | `.context/PROFILE.md` | No | Derived one-pager summary. Regenerated by `touch-domain --full`. |
| `MEMORY.md` | `.context/MEMORY.md` | No | Persistent distilled knowledge across sessions. Updated by distiller only. |
| `DECISIONS.md` | `.context/DECISIONS.md` | No | Structured decision log. Append-only. Updated by distiller only. |
| `STATE.md` | `.context/STATE.md` | No | Current status snapshot. Volatile, frequently updated. |
| `sessions/` | `.context/sessions/` | No | Session notes, memory drafts, transcripts. Permanent. |

## Load Order on Domain Entry

1. **Global governance** — `~/AGENTS.md`, `~/.claude/CLAUDE.md`
2. **Domain governance** — `<domain>/CLAUDE.md` (and `AGENTS.md` if present)
3. **Persona** — closest `persona.md` to the launch context
4. **Context files** — PROFILE.md → MEMORY.md → DECISIONS.md → STATE.md

---

## Self-Propagation

Global files (`~/AGENTS.md`, `~/.claude/CLAUDE.md`) serve as the authoritative templates for governance.

When `touch-domain` runs on a domain:
- If `CLAUDE.md` is missing at the repo root, the global version is detected and a domain copy is scaffolded (with a comment noting it was auto-generated from the global default)
- If `CLAUDE.md` exists, `touch-domain` can surface drift from the global default as a concern

This keeps governance consistent without requiring manual propagation.

## Retired

**`agent.md`** (previously at `.claude/agent.md`) — retired. Its responsibilities are now split:

| Former agent.md job | New home |
|---------------------|----------|
| Domain detection signal | `domain.yaml` |
| Persona / identity | `persona.md` |
| Context map | `persona.md` |
| Behavioural settings (model tier, distillation strategy) | `persona.md` |
| Project mechanics (build, test, conventions) | `CLAUDE.md` |
| Registry metadata | `domain.yaml` |

## Relationship to Ecosystem Standards

| Convention | Scope | Domain-toolkit stance |
|-----------|-------|----------------------|
| `AGENTS.md` | Universal agent instructions (Linux Foundation / AAIF) | Coexist. Used as global governance hook. `touch-domain` detects and surfaces. |
| `CLAUDE.md` | Claude Code project instructions (Anthropic) | Coexist. Used as both global and domain-level governance. Native Claude Code support. |
| `.cursorrules` | Cursor-specific agent rules | Tolerated. `touch-domain` can detect and surface. |
| `copilot-instructions.md` | GitHub Copilot instructions | Tolerated. Not actively managed. |
